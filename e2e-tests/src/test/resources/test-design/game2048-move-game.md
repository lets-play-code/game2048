# 移动 2048 游戏

## 被测接口

- `POST /api/games/{id}/move`

## 输入因子

| 因子 | 取值 |
| --- | --- |
| `direction` | `noop`、`left`、`right`、`up`、`down`、`escape` |
| 游戏是否已存在 | 不存在；已存在 |
| 棋盘形态 | 默认新局；仅移动不合并；已对齐不变化；可合并；合并出 2048；无路可走 |
| 终局标记 | 普通局；胜利局；失败局；已记录成绩的胜利局 |

## 输出因子

| 因子 | 取值 |
| --- | --- |
| HTTP 状态 | `200` |
| 返回游戏状态 | 默认新局；移动后新局面；重置后新局面；胜利局；失败局 |
| `score` / `scoreText` | 保持原值；合并后增加；重置为 `0` |
| `canMove` / `canSaveRecord` / `recordSaved` | 随普通局、胜利局、失败局切换 |
| auto 存档 | 每次请求后都刷新为当前局面 |

## 用例

| 用例名 | 关键输入 | 期望输出 |
| --- | --- | --- |
| 懒创建后 noop | `id` 不存在，`direction=noop` | 返回默认新局，并写入 auto 存档 |
| left 仅移动 | 单个 `4` 向左移动 | 分数不变，棋盘移动后写入 auto |
| right 已右对齐 | 第一行已右对齐 | 棋盘保持不变，auto 同步原局面 |
| up 已上对齐 | 第一列已上对齐 | 棋盘保持不变，auto 同步原局面 |
| down 已下对齐 | 第一列已下对齐 | 棋盘保持不变，auto 同步原局面 |
| left 发生合并 | 末行 `2,2,4,8` 左移 | 分数增加，棋盘合并并补新块，auto 刷新 |
| escape 重置 | 终局胜利局收到 `escape` | 清空终局标记并重置为默认新局 |
| 合并出 2048 | `1024,1024` 左移 | 返回胜利态、可保存成绩、消息数组正确 |
| 无路可走 | 满盘且不可合并时左移 | 返回失败态、可保存成绩、消息数组正确 |
