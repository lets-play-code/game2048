# 查询 2048 存档摘要

## 被测接口

- `GET /api/saves`

## 流程图

```text
[读取 SavedGames 表]
        |
        v
[按 auto → slot1 → slot2 → slot3 固定顺序遍历]
        |
        v
   {当前槽位有记录?}
    ├── Y → [hasData=true, 回显 score / savedAtUtc]
    └── N → [hasData=false, score=null, savedAtUtc=null]
        |
        v
[收集 4 个槽位摘要]
        |
        v
[返回 200 与摘要列表]
```

## 输入因子

| 因子 | 取值分析 |
| --- | --- |
| 固定槽位 | `auto`、`slot1`、`slot2`、`slot3`，顺序固定，必须全部出现在响应中 |
| 每个槽位是否存在存档记录 | 等价类：存在；不存在 |
| 已存在记录的 `score` | 等价类：任意非空整数；本次取代表值 `4`、`128` |
| 已存在记录的 `savedAtUtc` | 等价类：可解析的时间戳；本次取固定 UTC 字符串 `2026-01-01T00:00:00Z`、`2026-02-02T00:00:00Z` |

## 输出因子

| 因子 | 取值 |
| --- | --- |
| HTTP 状态 | `200` |
| 响应列表长度 | 固定为 `4` |
| 响应顺序 | 固定为 `auto`、`slot1`、`slot2`、`slot3` |
| 空槽位摘要 | `hasData=false`、`score=null`、`savedAtUtc=null` |
| 非空槽位摘要 | `hasData=true`，并回显该槽位记录的 `score` 与非空 `savedAtUtc` |
| 数据副作用 | `GET` 只读，`SavedGames` 表保持不变 |

## 最短路径与用例设计

| 用例名 | auto | slot1 | slot2 | slot3 | 期望输出 |
| --- | --- | --- | --- | --- | --- |
| 全部槽位为空 | 无记录 | 无记录 | 无记录 | 无记录 | 返回 4 个空摘要，顺序固定 |
| 稀疏存档按固定顺序返回 | `score=4` | 无记录 | `score=128` | 无记录 | 返回混合摘要：非空槽位回显数据，空槽位补 `null`，数据库不变 |

## 覆盖性检查

| 检查项 | 覆盖方式 |
| --- | --- |
| 所有代码路径已覆盖 | 用例 1 覆盖“无记录”路径；用例 2 覆盖“有记录”路径与混合列表拼装路径 |
| 输入因子的每个取值已使用 | 四个固定槽位都在响应中出现；“存在/不存在”两类槽位状态均被用例覆盖 |
| 条件分支每个判断逻辑点已覆盖 | `savedGame == null` 在用例 1 与用例 2 的空槽位上为真，在用例 2 的 `auto` 和 `slot2` 上为假 |
| 只读行为已覆盖 | 用例 2 额外断言 `SavedGames` 表内容保持不变 |
