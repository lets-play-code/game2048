# 读取 2048 游戏存档

## 被测接口

- `POST /api/games/{id}/load/{slotKey}`

## 输入因子

| 因子 | 取值 |
| --- | --- |
| `slotKey` | 非法槽位 `slot9`；合法但为空的 `slot3`；存在记录的 `slot2`；存在记录的 `auto` |
| 游戏是否已存在 | 不存在；已存在 |
| 存档记录 | 无记录；已有目标槽位记录 |
| 已有内存游戏状态 | 普通局面；被存档覆盖的旧局面 |

## 输出因子

| 因子 | 取值 |
| --- | --- |
| HTTP 状态 | `400` 或 `200` |
| 错误信息 | `Unknown save slot. (Parameter 'slotKey')`；`Save slot is empty.` |
| 返回游戏状态 | 默认新局；从指定槽位加载后的局面 |
| 存档表 | 保持不变 |

## 用例

| 用例名 | 关键输入 | 期望输出 |
| --- | --- | --- |
| 非法槽位 | `slotKey=slot9`，数据库已有其他槽位 | 返回 `400`，现有存档不变 |
| 空槽位懒创建 | `slotKey=slot3`，目标槽位为空，`id` 不存在 | 返回 `400`；随后 `GET` 同 id 可读到默认新局 |
| 不存在 id 加载手动槽位 | `slotKey=slot2`，目标槽位存在，`id` 不存在 | 返回 `200`，内存游戏按存档内容创建并加载 |
| auto 覆盖已存在游戏 | `slotKey=auto`，目标槽位存在，`id` 已存在旧局面 | 返回 `200`，响应与后续 `GET` 都等于 auto 槽位内容 |
